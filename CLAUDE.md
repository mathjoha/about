# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

This is a personal academic website for Mathias Johansson, built with [Quarto](https://quarto.org/). It showcases CV, publications, datasets, code projects, and teaching experience. Requires only Quarto (with TinyTeX for PDF) and Python 3 ‚Äî no package.json or requirements.txt.

## Commands

- **Preview locally**: `quarto preview` (runs on port 5057)
- **Build site**: `quarto render` (outputs to `_site/`)
- **Publish**: Automatic via GitHub Actions on push to `main`
- **Generate PDF CV**: `python3 generate_cv_pdf.py` then `quarto render cv_for_pdf.qmd --to pdf`

## Generated Files ‚Äî Do Not Edit Directly

These files are generated by the pre-render pipeline. Edit the sources instead:

- `cv/full.qmd` ‚Äî generated from `cv/short.qmd` by `prepare_full.py`
- `cv_for_pdf.qmd` ‚Äî generated by `generate_cv_pdf.py`
- `CV_MJ.latest.pdf` ‚Äî rendered from `cv_for_pdf.qmd`

## Architecture

### Pre-render Script

`prepare_full.py` runs as a Quarto pre-render script (configured in `_quarto.yml`). It copies `cv/short.qmd` to `cv/full.qmd` with `max-items: 3` removed, creating a full CV version without item limits. It then calls `generate_cv_pdf.py` and renders the PDF. Uses `QUARTO_PRERENDER_RUNNING` env var to prevent infinite recursion.

### PDF CV Generator

`generate_cv_pdf.py` creates a flat `cv_for_pdf.qmd` file from CV data (YAML files and QMD frontmatter). This bypasses Quarto's listings feature which doesn't work with PDF output. The generated file includes LaTeX header/footer configuration with name, date, URL, and page numbers. Includes a custom YAML parser fallback when PyYAML is not installed (handles multiline strings and simple lists).

### Grant Application CV

`generate_grant_pdf.py` produces a compact `cv_grant.qmd` for grant applications. It imports shared utilities from `generate_cv_pdf.py` and merges private experience details from `cv/experience.private.yaml` (gitignored). Both the script and its outputs (`cv_grant.qmd`, `cv_grant.pdf`) are gitignored.

### Content Structure

- **CV data**: YAML files in `cv/` (`experience.yaml`, `education.yaml`, `teaching.yaml`, `skills.yaml`, `presentations.yaml`, `visualizations.yaml`)
- **Projects**: Individual `.qmd` files in `cv/project/`
- **Publications**: `.qmd` files in `pub/`
- **Datasets**: `.qmd` files in `cv/dataset/`
- **Code projects**: `.qmd` files in `code/`
- **Blog posts**: `.qmd` files in `YYYY/MM/` directories (e.g. `2025/05/`), indexed by `index.qmd` with glob `'20[23][0-9]/??/*qmd'`

### Content Frontmatter Conventions

Publications (`pub/*.qmd`) use these fields: `title`, `subtitle`, `Type`, `Journal`, `authors`, `date`, `doi`, `file`, `url`, `repo`, `abstract`. Author self-emphasis convention: `**<u>Mathias Johansson</u>**`. Titles can embed shortcodes and links: `[{{< ai open-access >}} Title](/pub/file.qmd)`.

Datasets (`cv/dataset/*.qmd`) and code projects (`code/*.qmd`) use: `title`, `subtitle`, `date`, plus optional links.

### Listing Sort Order

- Education: `sort-order` field, then start/end dates descending
- Projects: `active` descending, then `start` ascending
- Teaching: `year` descending
- Publications, datasets, code: `date` descending

### Listing Templates

Custom EJS templates in `js/` control how listings render:
- `exp.ejs` - Experience entries
- `edu.ejs` - Education entries
- `pro.ejs` - Project entries
- `skills.ejs` - Skills & languages
- `pres.ejs` - Presentations
- `vis.ejs` - Explorations/visualizations

### Quarto Extensions & Icon Shortcodes

Three icon shortcode patterns are available:
- `{{< ai orcid >}}` ‚Äî academicons (`schochastics/academicons`)
- `{{< iconify bi database >}}` ‚Äî generic icons (`mcanouil/iconify`)
- `{{< devicon python-plain >}}` ‚Äî developer icons (`schochastics/devicons`)
